ШАГ 1: НА WINDOWS - Перенесите проект на Linux сервер

Откройте PowerShell в папке c:\Я\ICIDS\serv_1\StrikeballServer

    .\transfer-to-linux.ps1 -Server 192.168.1.100 -User root

    # Замените 192.168.1.100 на IP вашего сервера
    # Система попросит пароль

═════════════════════════════════════════════════════════════════════════════════

ШАГ 2: НА LINUX - Установите сервер

SSH на сервер и запустите:

    cd /opt/StrikeballServer
    chmod +x deploy.sh
    sudo ./deploy.sh

    # Скрипт сам установит всё и запустит сервис

ШАГ 3: ПРОВЕРЬТЕ ЧТО РАБОТАЕТ

✨ ЧТО ПРОИСХОДИТ ПРИ РАЗВЕРТЫВАНИИ

deploy.sh автоматически:

1. Установит зависимости (curl, wget, postgresql)
2. Скачает и установит .NET 8 Runtime
3. Создаст пользователя strikeball
4. Создаст директории /opt/strikeball/server
5. Установит и настроит PostgreSQL
6. Создаст базу данных strikeballdb
7. Скомпилирует приложение в Release режиме
8. Установит systemd сервис
9. Запустит сервис
10. Покажет статус и логи

ИТОГО: 3-5 минут и сервер работает!

❓ ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ

Q: Какой IP использовать?
A: IP вашего Linux сервера в локальной сети. Например: 192.168.1.100

Q: Можно ли использовать Docker вместо Linux?
A: Да, просто запустите: docker-compose up -d

Q: Сколько маяков может обработать?
A: С PostgreSQL: 1000+ измерений в секунду, неограниченное количество маяков

Q: Можно ли добавить аутентификацию?
A: Да, отредактируйте Program.cs (добавить JWT middleware)

Q: Как очистить старые данные?
A: Смотрите в LINUX_DEPLOYMENT_STEP_BY_STEP.sh раздел "Отладка"
